# 9.3 Directory Descriptions

## Root Files

### README.md

Provides a complete overview of the project, installation instructions, technologies used, project structure, and deployment guide.

### .gitignore

Specifies files and folders that Git should ignore during version control.

---

## 01_Project_Documentation

Contains all planning, analysis, design, management, and supporting documentation for the project.

### 01_Project_Initiation

Stores all documents produced during Phase 0, including project overview, objectives, scope, requirements, and development standards.

### 02_Brand_Strategy

Contains brand identity documents such as logo concepts, colour palette, typography decisions, and visual branding guidelines.

### 03_Information_Architecture

Documents the website's content structure, navigation hierarchy, sitemap, and user flow.

### 04_UI_UX_Design

Contains wireframes, design mock-ups, UI components, and UX documentation.

### 05_Project_Management

Stores project schedules, milestones, meeting notes (if any), change logs, risk registers, and related management documents.

---

## 02_Source_Code

Contains all files required to build the website.

### css

Stores all Cascading Style Sheets.

### js

Stores JavaScript files responsible for interactive functionality.

### images

Stores optimized images used directly by the website.

### index.html

Acts as the primary entry point for the website.

---

## 03_Assets

Stores reusable project resources that are not directly part of the application's source code.

### Images

Original high-resolution images and graphics.

### Icons

Icon libraries and custom icons.

# 9.4 Folder Organization Principles

The project structure shall follow these principles:

- Logical separation of concerns.
- Clear and descriptive naming conventions.
- Minimal folder nesting.
- Scalability for future expansion.
- Consistency throughout the project.
- Ease of navigation for developers.

---

# 9.5 Future Expansion

As the project evolves, additional directories may be introduced for testing, automation, reusable UI components, or advanced front-end frameworks. Such additions shall preserve the organizational principles established in this document.

---

# 9.6 Folder Structure Summary

The directory structure of the Professional Portfolio Website has been designed to support clean organization, efficient development, and long-term maintainability. Every folder has a defined responsibility, ensuring that project assets, documentation, and source code remain well organized as the website grows.
