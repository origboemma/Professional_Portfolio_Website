# 1. Purpose

The Project Change Log records all significant changes made throughout the lifecycle of the Professional Portfolio Website.

Unlike the Project Decisions Log, which explains why strategic decisions were made, the Project Change Log records what was modified, when it was modified, why the modification was necessary, and its impact on the project.

This document supports project governance, traceability, and continuous improvement.

---

# 2. Scope

The Change Log records changes relating to:

- Project Scope
- Documentation
- Architecture
- Branding
- Information Architecture
- UI/UX Design
- Frontend Development
- Backend Development
- Testing
- Deployment
- Future Enhancements

Minor editorial corrections, spelling fixes, or formatting changes are generally excluded unless they affect the project's meaning.

---

# 3. Change Classification

Each change shall be classified as one of the following:

| Type | Description |
|------|-------------|
| Addition | New functionality or documentation introduced |
| Modification | Existing content updated or improved |
| Removal | Existing content removed |
| Correction | Error corrected without changing functionality |
| Enhancement | Improvement that increases quality or usability |
| Refactoring | Structural improvement without changing behaviour |

---

# 4. Change Status

| Status | Meaning |
|---------|---------|
| Planned | Approved but not yet implemented |
| In Progress | Currently being implemented |
| Completed | Successfully implemented |
| Cancelled | No longer required |

---

# 5. Change Register

| CHG-005 | 22 Jul 2026 | Documentation | Added Project Decisions Log to record strategic decisions. | Addition | Completed |

---

# 6. Change Impact Assessment

Before approving a major change, consider its impact on:

- Scope
- Schedule
- Documentation
- Development Effort
- User Experience
- Maintainability
- Performance
- Accessibility
- Future Scalability

Where applicable, the impact assessment should be recorded alongside the change request.

---

# 7. Change Approval Process

Every significant project change should follow this process:

1. Identify the proposed change.
2. Assess the reason for the change.
3. Evaluate the impact.
4. Approve or reject the change.
5. Implement the approved change.
6. Record the change in this log.
7. Update related project documentation if necessary.

---

# 8. Relationship to Other Documents

The Project Change Log should be maintained alongside:

- Project Decisions Log
- Project Roadmap
- Design Review Checklist
- Development Standards

Together, these documents provide complete governance throughout the project lifecycle.

---
