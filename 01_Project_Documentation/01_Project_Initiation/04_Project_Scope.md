# 4. Project Scope

## 4.1 Purpose of the Scope

The Project Scope defines the boundaries of the Professional Portfolio Website. It specifies the features, content, technologies, and deliverables that are included in the project, identifies what is intentionally excluded, establishes the project's operational boundaries, and outlines opportunities for future expansion. Clearly defining the scope helps maintain focus, control project complexity, and prevent unnecessary changes during development.

# 4.3 Out of Scope

The following items are intentionally excluded from Version 1.0 of this project:

- User registration and login
- User accounts and authentication
- Database management systems
- Backend development
- Content Management System (CMS)
- Blog functionality
- E-commerce features
- Online payment processing
- User comments and forums
- File upload functionality
- Real-time messaging
- Administrative dashboard
- Multi-user management
- Server-side APIs
- Dynamic data storage

These features may be considered in future versions but are not part of the current project.

---

# 4.4 Project Boundaries

The project is limited to the following boundaries:

- Front-end web application only.
- Static website architecture.
- HTML5, CSS3, and JavaScript technologies.
- Git for version control.
- GitHub for repository management.
- GitHub Pages for hosting.
- Personal professional portfolio only.
- Publicly accessible project information only.
- No collection or storage of visitor data.

---

# 4.5 Assumptions

The successful completion of this project assumes that:

- GitHub Pages remains available for hosting.
- Modern web browsers support the implemented technologies.
- Project repositories remain publicly accessible.
- Professional information remains accurate and up to date.
- Internet connectivity is available for accessing external resources.

---

# 4.6 Constraints

The project is subject to the following constraints:

- Static website architecture.
- No backend server.
- No relational database.
- Hosting limited to GitHub Pages.
- Development limited to front-end technologies.
- Project development will follow a phased approach with formal reviews and approvals.
