# 6. User Flows

## 6.1 Purpose

This document defines the primary user flows within the Professional Portfolio Website.

A user flow describes the sequence of pages and actions that visitors take to accomplish specific goals.

The objective is to create intuitive navigation paths that minimise effort while maximising engagement and understanding of Emmanuel Chukwuka Origbo's professional expertise.

---

# 6.2 User Flow Philosophy

Every user flow should:

- Be simple.
- Require minimal clicks.
- Maintain user orientation.
- Present information in a logical order.
- Encourage further exploration.
- End with a meaningful call-to-action.

Visitors should never feel lost or uncertain about the next step.

---

# 6.3 Primary User Goals

The website is designed to support several common visitor goals.

These include:

- Learn about Emmanuel.
- Review professional projects.
- Explore technical skills.
- Verify certifications.
- Understand career progression.
- Download the résumé.
- Contact Emmanuel.

Each goal is supported by a dedicated user flow.

# 6.10 Call-to-Action Integration

Each major page should include at least one clear call-to-action.

Examples include:

- View Project
- Explore Skills
- Read My Journey
- Download Resume
- View GitHub
- Contact Me

Calls-to-action should naturally guide visitors toward deeper engagement.

---

# 6.11 Exit Points

Visitors may leave the website from several pages.

To improve retention, each exit point should provide additional navigation options.

Examples include:

- Related Projects
- GitHub Profile
- LinkedIn Profile
- Resume Download
- Contact Form

These options encourage visitors to continue exploring rather than leaving immediately.
