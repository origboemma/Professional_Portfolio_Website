# 2. Website Structure

## 2.1 Purpose

This document defines the structural organisation of the Professional Portfolio Website.

It describes how pages, content, and user interactions are organised to create an intuitive, scalable, and professional digital experience.

The structure is designed to reflect Emmanuel Chukwuka Origbo's professional identity while guiding visitors through a logical journey from introduction to engagement.

---

# 2.2 Structural Philosophy

The Professional Portfolio Website is designed around a simple principle:

> **Introduce → Build Trust → Demonstrate Expertise → Encourage Engagement**

Every page exists to support one or more of these objectives.

Rather than presenting isolated information, the website tells a professional story that gradually builds confidence in Emmanuel's knowledge, skills, and practical experience.

---

# 2.3 Structural Model

The website follows a hybrid architecture.

## Level 1 – Executive Homepage

The homepage serves as an executive summary of the entire website.

It provides concise previews of:

- Professional Identity
- Professional Journey
- Featured Projects
- Skills
- Certifications
- Experience
- Contact Information

Its purpose is to help visitors understand Emmanuel's professional profile within the first minute of browsing.

---

## Level 2 – Dedicated Information Pages

Each major topic has its own dedicated page that expands upon the summaries presented on the homepage.

These include:

- About
- Professional Journey
- Projects
- Skills & Technologies
- Certifications
- Experience
- Resume
- Contact

Each page has a clearly defined purpose and avoids unnecessary overlap.

---

## Level 3 – Project Case Studies

Each professional project has its own dedicated page.

Rather than presenting projects as simple portfolio cards, every project is treated as a complete professional case study.

Each case study demonstrates:

- Business understanding
- Analytical thinking
- System design
- Technical implementation
- Business value
- Professional documentation

This structure reflects real-world consulting and enterprise project reporting.

---


This hierarchy keeps navigation simple while supporting unlimited future growth.

---

# 2.5 Information Flow

Visitors should naturally move through the website in the following sequence:

1. Discover Emmanuel.
2. Understand his professional background.
3. Explore completed projects.
4. Review technical skills.
5. Verify certifications.
6. Review professional experience.
7. Download the résumé.
8. Make contact.

The structure intentionally builds trust before requesting visitor action.

---
