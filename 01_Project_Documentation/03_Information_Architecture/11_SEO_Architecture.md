# 11. SEO Architecture

## 11.1 Purpose

This document defines the Search Engine Optimisation (SEO) architecture for the Professional Portfolio Website.

The objective is to maximise the visibility of the website on search engines while ensuring that visitors can easily discover Emmanuel Chukwuka Origbo's professional profile, projects, and expertise.

SEO should be integrated into the website architecture from the beginning rather than added after development.

---

# 11.2 SEO Objectives

The website aims to:

- Increase online visibility.
- Showcase professional expertise.
- Improve discoverability by recruiters and hiring managers.
- Support GitHub Pages indexing.
- Encourage organic traffic.
- Strengthen Emmanuel's professional digital presence.

---

# 11.3 Target Audience

SEO should focus on users searching for professionals in:

- Information Systems
- Data Analytics
- Business Analysis
- Data Engineering
- Analytics Engineering
- Business Intelligence
- SQL
- Python
- Power BI
- PostgreSQL

The content should answer real business and technical questions while demonstrating practical experience.

---

# 11.4 SEO Principles

The website should follow these principles:

- High-quality original content.
- Clear page hierarchy.
- Descriptive headings.
- Meaningful URLs.
- Fast loading pages.
- Mobile-friendly design.
- Accessible content.
- Internal linking between related pages.
- Regular content updates.

---

# 11.5 Page Titles

Every page should have a unique and descriptive title.

Examples:

Home

```
Emmanuel Chukwuka Origbo | Information Systems & Data Analytics Portfolio
```

Projects

```
Projects | Emmanuel Chukwuka Origbo
```

PrimeMart Project

```
PrimeMart Retail Sales Operations Reporting System | Case Study
```

About

```
About Emmanuel Chukwuka Origbo
```

---

# 11.6 Meta Descriptions

Every page should include a concise summary.

Example:

```
Professional portfolio of Emmanuel Chukwuka Origbo showcasing Information Systems, Data Analytics, Business Analysis, Data Engineering, Analytics Engineering, and software projects.
```

Meta descriptions should accurately reflect page content.

---

# 11.7 Heading Structure

Pages should use a logical heading hierarchy.

Example:

```
H1
    H2
        H3
```

Only one H1 heading should appear on each page.

Heading levels should never be skipped unnecessarily.

---

# 11.8 Internal Linking Strategy

Internal links should connect related content.

Examples:

- Skills → Related Projects
- Certifications → Skills
- Projects → Case Studies
- Case Studies → Related Projects
- About → Professional Journey
- Resume → Contact

This improves both navigation and search engine crawling.

---

# 11.9 Image Optimisation

Images should include:

- Descriptive filenames.
- Alternative text.
- Appropriate dimensions.
- Optimised file sizes.
- Modern image formats such as WebP where practical.

Project screenshots should clearly illustrate completed work.

---

# 11.10 Technical SEO

The website should include:

- sitemap.xml
- robots.txt
- Canonical URLs where applicable.
- Open Graph metadata.
- Twitter/X Card metadata.
- Favicon.
- HTTPS deployment through GitHub Pages.

These elements improve discoverability and social sharing.

---

# 11.11 Structured Data

Where appropriate, structured data should be added to help search engines understand the website.

Potential schema types include:

- Person
- Website
- WebPage
- CreativeWork
- Project
- BreadcrumbList

Structured data should accurately represent the website content.

---

# 11.12 Performance Considerations

SEO benefits from good performance.

The website should:

- Load quickly.
- Minimise unnecessary assets.
- Optimise images.
- Reduce JavaScript where possible.
- Follow responsive design principles.

---

# 11.13 Content Quality

Content should:

- Be original.
- Be professionally written.
- Demonstrate practical expertise.
- Remain accurate.
- Be updated as projects evolve.

The emphasis should be on quality rather than quantity.
