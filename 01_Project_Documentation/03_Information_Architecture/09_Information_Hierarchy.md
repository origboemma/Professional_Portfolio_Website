# 9. Information Hierarchy

## 9.1 Purpose

This document defines how information is prioritised and presented throughout the Professional Portfolio Website.

The objective is to ensure that visitors immediately see the most important information, followed by supporting details in a logical sequence. A clear information hierarchy improves readability, usability, and decision-making while helping visitors quickly understand Emmanuel Chukwuka Origbo's professional value.

---

# 9.2 Information Hierarchy Principles

The website follows the following principles:

- Present the most valuable information first.
- Reveal supporting information progressively.
- Maintain consistent information order across similar pages.
- Reduce unnecessary cognitive load.
- Emphasise business value before technical detail.
- Use visual design to reinforce information priority.

---

# 9.3 Homepage Information Hierarchy

The homepage should present information in the following order:

1. Hero Section
2. Professional Value Proposition
3. Featured Projects
4. Skills & Technologies
5. Certifications
6. Professional Journey Preview
7. Experience Highlights
8. Contact Call-to-Action
9. Footer

The homepage should answer three questions within the first few seconds:

- Who is Emmanuel?
- What problems can he solve?
- Why should the visitor continue exploring?

---

# 9.4 Project Page Information Hierarchy

Each project case study should present information in the following sequence:

1. Project Title
2. Executive Summary
3. Business Problem
4. Business Objectives
5. Stakeholders
6. Business Requirements
7. System Analysis
8. Solution Architecture
9. Technology Stack
10. Database Design (where applicable)
11. ETL Process (where applicable)
12. Implementation
13. Dashboard or Application Screenshots
14. Business Insights
15. Recommendations
16. Challenges Encountered
17. Lessons Learned
18. Future Enhancements
19. GitHub Repository
20. Related Projects

This order mirrors the lifecycle of a professional project and helps readers understand both the business context and the technical implementation.

---

# 9.5 About Page Information Hierarchy

The About page should follow this structure:

1. Professional Introduction
2. Mission
3. Vision
4. Core Values
5. Areas of Expertise
6. Professional Philosophy
7. Career Objectives

The emphasis should remain on professional identity rather than personal history.

---

# 9.6 Skills & Technologies Information Hierarchy

Information should be organised as follows:

1. Core Competencies
2. Technical Categories
3. Individual Technologies
4. Practical Experience
5. Related Projects

Visitors should immediately understand not only which technologies Emmanuel knows, but also how they have been applied in real projects.

---

# 9.7 Certifications Information Hierarchy

Each certification should display:

1. Certification Name
2. Issuing Organisation
3. Key Skills Acquired
4. Completion Year
5. Credential Link (where available)

The focus should remain on the knowledge gained rather than simply listing certificates.

---

# 9.8 Experience Information Hierarchy

Experience should be presented in this order:

1. Position
2. Organisation
3. Duration
4. Responsibilities
5. Achievements
6. Skills Demonstrated

This structure highlights impact rather than job titles alone.

---

# 9.9 Contact Page Information Hierarchy

The Contact page should prioritise:

1. Invitation to Connect
2. Contact Form
3. Email Address
4. LinkedIn Profile
5. GitHub Profile
6. Professional Availability

The objective is to make contacting Emmanuel straightforward and welcoming.

---

# 9.10 Visual Reinforcement

Information hierarchy should be reinforced through:

- Typography
- Heading levels
- White space
- Colour contrast
- Icons
- Cards
- Buttons
- Section spacing
- Responsive layouts

Visual design should always support—not replace—the logical structure of the content.

---

# 9.11 Consistency Guidelines

Information hierarchy should remain consistent across similar pages.

For example:

- All project pages should use the same section order.
- All certification entries should use the same layout.
- All skills categories should follow the same structure.

Consistency improves usability and makes the website easier to maintain.

# 9.13 Information Hierarchy Summary

The Information Hierarchy ensures that every page of the Professional Portfolio Website presents information in a logical, consistent, and user-focused manner. By prioritising business value, practical achievements, and professional evidence before supporting details, the website enables visitors to quickly understand Emmanuel Chukwuka Origbo's expertise and encourages deeper engagement.
