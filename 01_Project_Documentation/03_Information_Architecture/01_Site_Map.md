# 1. Site Map

## 1.1 Purpose

This document defines the high-level structure of the Professional Portfolio Website by identifying every major page and illustrating how they relate to one another.

The Site Map provides the foundation for navigation design, page hierarchy, content organisation, user experience, and future website expansion.

The website follows a hybrid architecture consisting of a storytelling homepage supported by dedicated pages for detailed professional content.

---

# 1.2 Website Architecture

The Professional Portfolio Website combines a modern scrolling homepage with dedicated pages for in-depth content.

The homepage introduces Emmanuel Chukwuka Origbo's professional brand, while secondary pages provide detailed information about projects, experience, certifications, skills, and other professional achievements.

This approach delivers an engaging user experience without sacrificing scalability or content depth.

# 1.4 Homepage Structure

The homepage serves as a professional landing page and provides a concise overview of Emmanuel's professional profile.

It consists of the following sections:

- Hero Section
- About Preview
- Professional Journey Preview
- Featured Projects
- Skills Overview
- Certifications Overview
- Experience Highlights
- Contact Call-to-Action
- Footer

Each section introduces visitors to the website while encouraging exploration of dedicated pages.

The main navigation menu will contain:

The navigation should remain simple, consistent, and available across every page.

---

# 1.6 Projects Architecture

The Projects section serves as the central showcase of Emmanuel's professional work.

Each project will have its own dedicated page structured as a professional case study.

Every project page should include:

- Executive Summary
- Business Problem
- Business Objectives
- Business Analysis
- System Analysis
- Solution Architecture
- Technology Stack
- Database Design (where applicable)
- ETL Pipeline (where applicable)
- Implementation
- Screenshots
- Results
- Lessons Learned
- GitHub Repository
- Live Demonstration (where available)

This approach allows visitors to understand both the technical implementation and the business value delivered by each project.

# 1.8 Website Depth

The website is intentionally designed with a shallow hierarchy.

Users should be able to reach important information:

- Within one click from the homepage whenever possible.
- Within no more than three clicks from any location.

This improves both usability and search engine indexing.
