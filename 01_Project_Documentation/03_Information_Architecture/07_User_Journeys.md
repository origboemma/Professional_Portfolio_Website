# 7. User Journeys

## 7.1 Purpose

This document describes the end-to-end experiences of different visitors to the Professional Portfolio Website.

Unlike User Flows, which define navigation paths, User Journeys describe the motivations, expectations, and objectives of visitors as they interact with the website.

The goal is to ensure that every visitor enjoys a purposeful, informative, and engaging experience.

---

# 7.2 Journey Philosophy

The website is designed around a simple progression:

**Discover → Understand → Trust → Engage**

Every page should contribute to moving visitors naturally through these four stages.

---

# 7.3 Primary Visitor Personas

The website is designed primarily for:

- Recruiters
- Hiring Managers
- Business Analysts
- Data Analytics Managers
- Information Systems Professionals
- Potential Clients
- Project Collaborators
- Academic Supervisors
- Fellow Technology Professionals

Each visitor type has different goals but should experience a consistent level of professionalism.

---

# 7.9 Shared Journey Principles

Regardless of visitor type, every journey should provide:

- Clear navigation
- Logical progression
- Professional presentation
- Consistent branding
- High-quality documentation
- Evidence-based project showcases
- Easy access to contact information

---

# 7.10 Journey Success Indicators

A successful visitor journey is one in which the visitor:

- Understands Emmanuel's professional identity.
- Recognises his technical and analytical capabilities.
- Sees evidence through real projects.
- Feels confident in his ability to deliver value.
- Knows how to make contact.
