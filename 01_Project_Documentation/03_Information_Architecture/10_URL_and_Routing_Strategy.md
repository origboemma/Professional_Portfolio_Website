# 10. URL & Routing Strategy

## 10.1 Purpose

This document defines the URL structure and routing strategy for the Professional Portfolio Website.

The objective is to create URLs that are simple, meaningful, user-friendly, search-engine friendly, and scalable as the portfolio grows.

A consistent routing strategy also improves navigation, maintainability, and future migration to other web technologies.

---

# 10.2 URL Design Principles

All URLs should:

- Be short and descriptive.
- Use lowercase letters only.
- Use hyphens (-) to separate words.
- Avoid spaces and special characters.
- Remain stable over time.
- Reflect the website hierarchy.
- Be easy to read and share.

# 10.6 Internal Linking

Pages should be connected through descriptive internal links.

Examples:

- Homepage → Featured Projects
- Projects → Individual Case Studies
- Case Studies → Related Projects
- Skills → Related Projects
- Certifications → Relevant Skills
- Resume → Contact

Internal linking improves navigation and supports search engine indexing.

---

# 10.7 Routing Consistency

All future pages should follow the established conventions.

Examples:

Correct:

```
/projects/customer-segmentation-analysis
```

Incorrect:

```
/Projects/CustomerSegmentation
```

Consistency improves usability and simplifies maintenance.

---

# 10.8 Broken Link Prevention

To minimise broken links:

- Avoid changing published URLs.
- Use consistent naming conventions.
- Update internal links whenever new pages are added.
- Test all navigation before deployment.
