# 1. Purpose

This document defines the quality management framework for the Professional Portfolio Website.

It establishes the quality objectives, standards, review processes, testing activities, acceptance criteria, and continuous improvement practices that will be applied throughout the software development lifecycle.

The purpose of this plan is to ensure that every project deliverable satisfies agreed quality requirements before progressing to the next phase.

---

# 2. Quality Objectives

The objectives of quality management are to:

- Deliver a professional and reliable website.
- Produce maintainable, readable, and scalable code.
- Ensure documentation accuracy.
- Improve user experience.
- Minimise defects.
- Ensure accessibility compliance.
- Optimise performance.
- Promote continuous improvement.

---

# 3. Quality Principles

Project quality is guided by the following principles:

- Prevention over correction.
- Quality at every phase.
- Documentation-driven development.
- Continuous verification.
- Evidence-based reviews.
- User-centred design.
- Continuous improvement.

Quality is everyone's responsibility throughout the project lifecycle.

---

# 4. Quality Standards

The project will comply with the following standards:

## Documentation Quality

Documentation must:

- Be complete.
- Be accurate.
- Be well structured.
- Use consistent terminology.
- Maintain version history.
- Be reviewed before approval.

---

## Design Quality

The UI/UX design must:

- Be visually consistent.
- Follow the design system.
- Support responsive layouts.
- Meet accessibility requirements.
- Provide an intuitive user experience.

---

## Code Quality

Source code must:

- Follow agreed coding standards.
- Be modular.
- Be reusable.
- Be maintainable.
- Be properly documented.
- Avoid unnecessary complexity.

---

## User Experience Quality

The website should:

- Be easy to navigate.
- Load quickly.
- Provide consistent interactions.
- Work across modern browsers.
- Support desktop, tablet, and mobile devices.

---

# 5. Quality Assurance

Quality Assurance (QA) focuses on preventing defects by ensuring that the correct processes are followed.

QA activities include:

- Documentation reviews.
- Architecture reviews.
- Design reviews.
- Code reviews.
- Standards compliance checks.
- Process audits.

QA is performed throughout the project rather than only at the end.

---

# 6. Quality Control

Quality Control (QC) focuses on verifying completed deliverables.

QC activities include:

- Functional testing.
- Responsive testing.
- Accessibility testing.
- Performance testing.
- Visual inspection.
- Defect verification.

Quality Control confirms that deliverables meet defined acceptance criteria.

---

# 7. Review Process

Each major deliverable should follow this review workflow:

1. Preparation.
2. Self-review.
3. Technical review.
4. Quality review.
5. Approval.
6. Baseline update.

Only approved deliverables should proceed to implementation.

---

# 8. Testing Standards

The following testing activities will be performed:

### Functional Testing

Verify that all features operate as expected.

---

### Responsive Testing

Verify correct behaviour across different screen sizes.

---

### Browser Compatibility Testing

Verify consistent behaviour across supported browsers.

---

### Accessibility Testing

Verify compliance with recognised accessibility practices.

---

### Performance Testing

Measure loading speed, responsiveness, and optimisation.

---

### Regression Testing

Confirm that new changes do not introduce defects into existing functionality.

---

# 9. Acceptance Criteria

A deliverable is accepted when:

- All planned requirements are implemented.
- Reviews have been completed.
- Quality standards have been satisfied.
- Critical defects have been resolved.
- Documentation has been updated.
- Approval has been granted.

Acceptance should be documented before proceeding.

---

# 10. Defect Management

Defects should be managed using the following process:

1. Identify.
2. Record.
3. Prioritise.
4. Assign.
5. Correct.
6. Verify.
7. Close.

Each defect should remain traceable until resolution.

---

# 11. Continuous Improvement

Quality should continually improve through:

- Lessons learned.
- Review feedback.
- Retrospectives.
- Refactoring.
- Documentation improvements.
- Performance optimisation.

Continuous improvement should be incorporated into future project phases.
