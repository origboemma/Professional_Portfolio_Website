# 1. Purpose

This document defines the stakeholder management approach for the Professional Portfolio Website.

It identifies project stakeholders, analyses their interests and influence, establishes engagement strategies, and defines communication requirements to ensure that stakeholder expectations remain aligned with project objectives throughout the software development lifecycle.

---

# 2. Objectives

The Stakeholder Management Plan aims to:

- Identify all project stakeholders.
- Understand stakeholder expectations.
- Define stakeholder responsibilities.
- Promote effective collaboration.
- Improve decision-making.
- Reduce stakeholder-related risks.
- Support long-term project success.

---

# 3. Stakeholder Management Principles

The project follows these stakeholder management principles:

- Early stakeholder identification.
- Open and transparent communication.
- Active engagement.
- Respect for stakeholder needs.
- Continuous feedback.
- Accountability.
- Continuous improvement.

Stakeholder engagement should evolve as the project progresses.

---

# 4. Stakeholder Identification

The Professional Portfolio Website involves the following stakeholder groups:

### Internal Stakeholders

- Project Owner
- Project Manager
- Business Analyst
- System Analyst
- UI/UX Designer
- Software Engineer
- Quality Assurance Engineer

---

### External Stakeholders

- Recruiters
- Hiring Managers
- Potential Employers
- Clients
- Project Reviewers
- Open Source Contributors
- Technical Mentors
- Future Collaborators
- Portfolio Visitors

Although several internal roles are currently performed by the same individual, they are documented separately to reflect standard software engineering practice.

# 6. Stakeholder Analysis

Stakeholders are analysed according to:

- Level of Interest
- Level of Influence
- Communication Needs
- Engagement Level
- Decision-Making Authority

Understanding these factors helps determine the most appropriate engagement strategy.

---

# 7. Stakeholder Engagement Strategy

The project adopts the following engagement approach:

### High Influence / High Interest

Examples:

- Project Owner
- Project Manager

Strategy:

Manage closely through regular reviews, approvals, and decision-making.

---

### High Influence / Low Interest

Examples:

- Hiring Managers (during recruitment)

Strategy:

Keep satisfied through concise, high-quality project documentation and demonstrations.

---

### Low Influence / High Interest

Examples:

- Recruiters
- Portfolio Visitors
- Future Collaborators

Strategy:

Keep informed with clear documentation, project updates, and accessible information.

---

### Low Influence / Low Interest

Examples:

- General visitors

Strategy:

Monitor engagement and provide relevant public information.

---

# 8. Stakeholder Communication Requirements

Stakeholders require different types of information.

| Stakeholder | Information Required | Frequency |
|-------------|----------------------|-----------|
| Project Owner | Overall progress, approvals, risks | At major milestones |
| Project Manager | Schedule, deliverables, issues | Weekly or phase completion |
| Software Engineer | Technical requirements, architecture updates | As required |
| QA Engineer | Test results, quality findings | Before each release |
| Recruiters | Completed portfolio, projects, skills | When portfolio is updated |
| Hiring Managers | Demonstrations, documentation, GitHub repository | During recruitment |
| Clients | Project capabilities and outcomes | On request |

---

# 9. Stakeholder Responsibilities

Stakeholders are expected to:

- Participate in reviews where appropriate.
- Provide timely feedback.
- Support project objectives.
- Follow governance processes.
- Respect documentation standards.
- Communicate issues promptly.

---

# 10. Stakeholder Review

Stakeholder analysis should be reviewed:

- At the completion of each project phase.
- When new stakeholders are introduced.
- When project objectives change.
- Before major releases.
- Following significant project changes.

Reviews help ensure continued alignment with stakeholder expectations.
