# 1. Purpose

This document defines how colour will be applied throughout the Professional Portfolio Website.

The objective is to ensure that colour supports usability, readability, accessibility, branding, and visual consistency rather than serving as decoration.

---

# 2. Colour Philosophy

Colours should:

- Strengthen the professional brand.
- Improve readability.
- Guide user attention.
- Communicate meaning.
- Maintain consistency.
- Support accessibility.

Colour should always enhance the user experience.

---

# 3. Colour Palette

The website adopts the colour palette established during the Brand Strategy phase.

The palette consists of:

| Colour Group | Purpose |
|--------------|---------|
| Primary | Brand identity and key interface elements |
| Secondary | Supporting interface elements |
| Accent | Highlights and calls-to-action |
| Neutral | Backgrounds, borders, and text |
| Success | Positive feedback |
| Warning | Important notices |
| Error | Validation and error states |

---

# 4. Colour Usage Principles

Colours should be applied consistently.

Examples include:

- Primary colours for key actions.
- Accent colours for emphasis.
- Neutral colours for backgrounds.
- Success colours for confirmation messages.
- Warning colours for cautionary information.
- Error colours for validation feedback.

No colour should be used without a clear purpose.

---

# 5. Interface Colour Mapping

| Interface Element | Colour Role |
|-------------------|-------------|
| Navigation Bar | Primary / Neutral |
| Hero Section | Primary |
| Buttons | Primary / Accent |
| Project Cards | Neutral |
| Skill Cards | Neutral |
| Section Backgrounds | Neutral |
| Footer | Primary |
| Links | Accent |
| Form Inputs | Neutral |
| Alerts | Success / Warning / Error |

---

# 6. Text Colour Guidelines

Text should maintain sufficient contrast against its background.

General rules:

- Primary text on light backgrounds.
- Light text on dark backgrounds.
- Muted text for supporting information.
- Accent colour only for emphasis.

Text colour should never reduce readability.

---

# 7. Interactive States

Interactive elements should clearly communicate their status.

Each interactive component should support:

- Default
- Hover
- Focus
- Active
- Disabled

Users should always receive visual feedback.

---

# 8. Accessibility Requirements

Colour implementation shall comply with recognised accessibility practices.

Requirements include:

- Sufficient contrast ratios.
- Colour should not be the only means of conveying information.
- Visible keyboard focus indicators.
- Readable text across all devices.

Accessibility takes precedence over aesthetics.

---

# 9. Responsive Colour Behaviour

The colour palette should remain consistent across:

- Desktop
- Laptop
- Tablet
- Mobile

Responsive layouts should not introduce inconsistent colour usage.

---

# 10. Future Theme Support

The design should allow for future expansion to support:

- Light Theme
- Dark Theme
- High Contrast Theme

The colour system should be flexible enough to accommodate additional themes without requiring component redesign.

---

# 11. Frontend Implementation

During frontend development, colours should be implemented using reusable design tokens.

Examples include:

- CSS Custom Properties
- Tailwind CSS Theme Configuration
- Component Variables

Hard-coded colour values should be avoided wherever possible.
