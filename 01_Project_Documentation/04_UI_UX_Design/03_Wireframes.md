# 4. Wireframes

## 4.1 Purpose

This document defines the wireframing strategy for the Professional Portfolio Website.

Wireframes provide the structural blueprint for each page before visual styling and frontend development begin.

The objective is to validate page layouts, information placement, navigation flow, and component organisation while remaining independent of colours, imagery, and final visual styling.

---

# 4.2 Wireframing Philosophy

Wireframes should answer four fundamental questions:

- What information appears on the page?
- Where is the information positioned?
- How do visitors move through the page?
- What actions should visitors take?

At this stage, visual aesthetics are secondary to usability and structure.

---

# 4.3 Three-Level Wireframe Process

The website follows a progressive wireframing approach.

## Level 1 – Low-Fidelity Wireframes

Purpose:

Validate page structure.

Characteristics:

- Simple boxes
- Placeholder text
- No colours
- No images
- No typography styling
- Focus on layout only

---

## Level 2 – Mid-Fidelity Wireframes

Purpose:

Validate interface organisation.

Characteristics:

- Real headings
- Buttons
- Cards
- Spacing
- Typography hierarchy
- Navigation placement

Colours and branding remain minimal.

---

## Level 3 – High-Fidelity Mockups

Purpose:

Represent the final website before development.

Characteristics:

- Brand colours
- Typography
- Icons
- Images
- Animations (where appropriate)
- Complete design system
- Responsive layouts

These mockups should closely resemble the final implementation.

---

# 4.4 Homepage Wireframe

The homepage should contain:

1. Hero Section
2. Professional Value Proposition
3. Featured Projects
4. Skills Overview
5. Certifications
6. Professional Journey
7. Experience Highlights
8. Contact Call-to-Action
9. Footer

Each section should naturally guide visitors to the next.

---

# 4.5 Internal Page Wireframes

All internal pages should follow a consistent structure:

- Header
- Navigation
- Page Banner
- Main Content
- Supporting Content
- Recommended Next Step
- Footer

Consistency improves usability and reduces the learning curve for visitors.

---

# 4.6 Project Page Wireframes

Every project case study should use the same layout:

- Project Hero
- Executive Summary
- Business Problem
- Business Objectives
- Stakeholders
- System Analysis
- Solution Design
- Technology Stack
- Database Design
- ETL Pipeline
- Implementation
- Dashboard or Application Screenshots
- Business Insights
- Recommendations
- Challenges
- Lessons Learned
- GitHub Repository
- Related Projects
- Contact Call-to-Action

This creates a consistent reading experience across all projects.

---

# 4.7 Responsive Wireframes

Wireframes should be created for:

- Desktop
- Tablet
- Mobile

Layouts should adapt while preserving the established information hierarchy.

---

# 4.8 Navigation Validation

Each wireframe should confirm:

- Clear page hierarchy
- Logical navigation
- Visible calls-to-action
- Easy access to important content

Users should always know where they are and what to do next.

---

# 4.9 Component Placement

Wireframes should identify the placement of:

- Navigation Bar
- Hero Banner
- Cards
- Buttons
- Timelines
- Forms
- Footer
- Project Galleries
- Contact Sections

Component placement should remain consistent across similar pages.

---

# 4.10 Wireframe Review Criteria

Every wireframe should be evaluated against the following questions:

- Is the layout easy to understand?
- Does the content follow the Information Hierarchy?
- Are calls-to-action clearly visible?
- Is navigation intuitive?
- Does the layout support responsiveness?
- Does the page reinforce Emmanuel's professional value?

---

# 4.11 Relationship to Other Documents

The Wireframes document builds upon:

- UI/UX Framework
- Design System
- Layout System
- Information Architecture

It serves as the bridge between planning and visual design.

---

# 4.12 Wireframes Summary

The Wireframes establish the structural blueprint for the Professional Portfolio Website. By progressing from low-fidelity layouts to high-fidelity mockups, the project validates usability, content organisation, and navigation before frontend development begins, reducing redesign effort and ensuring a polished final product.
