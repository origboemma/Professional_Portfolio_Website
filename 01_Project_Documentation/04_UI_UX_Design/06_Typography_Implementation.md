# 1. Purpose

This document defines the typography standards for the Professional Portfolio Website.

Typography is a core element of the user interface. It affects readability, accessibility, visual hierarchy, and the overall perception of quality. The objective is to create a consistent and professional reading experience across every page and device.

---

# 2. Typography Philosophy

Typography should:

- Prioritise readability.
- Create a clear information hierarchy.
- Reinforce the professional brand.
- Remain consistent throughout the website.
- Support accessibility.
- Scale effectively across all screen sizes.

The typography should feel modern, clean, and business-focused.

---

# 3. Font Strategy

The website will use a limited set of font families.

| Font Role | Purpose |
|-----------|---------|
| Primary Font | Body text and general interface content |
| Display Font | Hero headings and major page titles (optional if different from the primary font) |
| Monospace Font | Code snippets, technical references, and terminal examples |

Using too many fonts should be avoided to maintain visual consistency.

---

# 4. Typography Hierarchy

The website uses a structured heading hierarchy.

| Element | Purpose |
|---------|---------|
| H1 | Main page title |
| H2 | Major section heading |
| H3 | Section subsection |
| H4 | Component heading |
| H5 | Minor heading |
| H6 | Supporting heading |
| Body Large | Important descriptive text |
| Body | Standard paragraph text |
| Small | Supporting information |
| Caption | Labels, metadata, and notes |

The hierarchy should remain consistent across all pages.

---

# 5. Text Alignment

General alignment rules:

- Left-align body content.
- Centre-align hero headings where appropriate.
- Align lists consistently.
- Avoid justified text for long paragraphs.

Alignment should improve readability rather than create visual symmetry.

---

# 6. Line Length and Spacing

Readable typography requires balanced spacing.

Guidelines include:

- Comfortable line lengths.
- Consistent line height.
- Appropriate paragraph spacing.
- Adequate separation between headings and body text.

Text should never appear cramped or excessively spaced.

---

# 7. Font Weights

Different font weights should communicate hierarchy.

Examples:

| Weight | Typical Use |
|---------|-------------|
| Light | Decorative or large display text |
| Regular | Standard body text |
| Medium | Labels and navigation |
| Semi-Bold | Subheadings |
| Bold | Primary headings and emphasis |

Weight should reinforce structure rather than replace it.

---

# 8. Responsive Typography

Typography should adapt to:

- Desktop
- Laptop
- Tablet
- Mobile

Font sizes should scale while preserving the established hierarchy and readability.

---

# 9. Accessibility Requirements

Typography shall support accessibility by ensuring:

- Sufficient font sizes.
- Clear contrast with backgrounds.
- Readable line spacing.
- Logical heading hierarchy.
- Scalable text without loss of usability.

Typography should support users with diverse visual abilities.

---

# 10. Frontend Implementation

Typography should be implemented using reusable design tokens.

Examples include:

- Font family tokens.
- Font size tokens.
- Font weight tokens.
- Line height tokens.
- Letter spacing tokens.

Avoid hard-coding typography values within individual components.
